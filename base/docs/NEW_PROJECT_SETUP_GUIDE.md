# New Project Setup Guide: "Monkey" Example

## Overview

Setting up a new project involves **two separate concerns**:
1. **OpenCode Configuration** (`.OpenCode` submodule) - AI assistant context and rules
2. **Boilerplate Code** (backend, frontend, etc.) - Actual application code

---

## Available Commands

| Command | Purpose | What it creates |
|---------|---------|-----------------|
| `/new-project` | **One-command setup** (recommended) | Everything: `.pi/`, boilerplate, docker-compose |
| `/initialize-OpenCode` | OpenCode configuration only | `.pi/` submodule with AI context |
| `/create-mono-repo` | Boilerplate code only | `backend/`, `frontend/`, etc. |

---

## Option A: One-Command Setup (Recommended)

```bash
mkdir monkey && cd monkey
/new-project Monkey
```

This single command will:
1. Ask for your tech stack (backend + frontend choices)
2. Create `CoFixer/monkey-OpenCode` repo on GitHub
3. Set up `.pi/` submodule with framework patterns
4. Clone selected boilerplate repos
5. Generate `docker-compose.yml`
6. Create `.project/` documentation structure
7. Make initial commit

**Done!** Your complete project is ready.

---

## Option B: Step-by-Step Setup

Use this if you need more control or only want part of the setup.

### Step 1: Create Empty Project Directory

```bash
mkdir monkey-project
cd monkey-project
git init
```

### Step 2: Run `/initialize-OpenCode` (OpenCode config)

```
/initialize-OpenCode Monkey
```

**What happens:**

1. **Asks for tech stack** via interactive prompts:
   - Backend: NestJS
   - Frontend: React Web, React Native, or both

2. **Creates a GitHub repo**: `CoFixer/monkey-OpenCode`
   - This holds all OpenCode configuration

3. **Adds framework submodules** based on your choices:
   ```
   monkey-OpenCode/
   ├── base/           → OpenCode-base (shared agents, hooks)
   ├── nestjs/         → OpenCode-nestjs (NestJS patterns)
   ├── react/          → OpenCode-react (React patterns)
   ├── agents/         (project-specific overrides)
   ├── hooks/          (project-specific hooks)
   ├── skills/         (skill-rules.json)
   ├── commands        → base/commands (symlink)
   └── settings.json
   ```

4. **Adds `.OpenCode` submodule** to your project:
   ```bash
   git submodule add https://github.com/CoFixer/monkey-OpenCode.git .OpenCode
   ```

**Result**: Your project now has OpenCode context but **NO application code yet**.

---

### Step 3: Run `/create-mono-repo` (Boilerplate Code)

```
/create-mono-repo
```

**What happens:**

1. **Interactively asks** for each repository to add:

   | Prompt | Example Answer |
   |--------|----------------|
   | GitHub URL? | `https://github.com/CoFixer/nestjs-starter-kit` |
   | Branch? | `main` |
   | Folder name? | `backend` |
   | Add another? | Yes |

   Repeat for each boilerplate:
   - `nestjs-starter-kit` → `backend/`
   - `react-starter-kit` → `frontend/`
   - `react-starter-kit` → `frontend-dashboard/` (same repo, different folder)
   - `react-native-starter-kit` → `mobile/`

2. **Clones each repo** into the specified folder

3. **Removes Docker files** from subfolders (to avoid conflicts)

4. **Removes `.git` directories** (so you have one unified git history)

5. **Generates root `docker-compose.yml`** for orchestration

**Result**: Your project now has actual application code.

## Final Project Structure

After both commands, "Monkey" project looks like:

```
monkey-project/
├── .pi/                    ← OpenCode config (from /initialize)
│   ├── base/                   → OpenCode-base submodule
│   ├── nestjs/                 → OpenCode-nestjs submodule
│   ├── react/                  → OpenCode-react submodule
│   ├── agents/                 (project-specific)
│   ├── hooks/                  (project-specific)
│   ├── skills/
│   │   └── skill-rules.json
│   ├── commands                → base/commands
│   └── settings.json
│
├── backend/                    ← NestJS app (from /create-mono-repo)
│   ├── src/
│   ├── package.json
│   └── ...
│
├── frontend/                   ← React app (from /create-mono-repo)
│   ├── src/
│   ├── package.json
│   └── ...
│
├── frontend-dashboard/         ← Another React app (optional)
│   └── ...
│
├── mobile/                     ← React Native app (optional)
│   └── ...
│
├── docker-compose.yml          ← Generated by /create-mono-repo
├── .gitignore
└── .gitmodules                 ← Tracks .OpenCode submodule
```

---

## What Happens to `.project`?

The `.project/` folder in this starter kit is **project-specific documentation** that:
- Contains plans, memory, and project knowledge
- Is NOT part of the `.OpenCode` submodule system
- Would be created fresh for your "Monkey" project

For a new project, you would create your own `.project/` structure:
```
monkey-project/
└── .project/
    ├── plans/              ← Implementation plans
    ├── memory/             ← Context for OpenCode
    └── docs/               ← Project documentation
```

## Example Workflows

### Quick Start (Recommended)

```bash
mkdir monkey && cd monkey
/new-project Monkey
# → Select: NestJS + React Web
# → Everything is set up automatically!
```

### Manual/Granular Control

```bash
# 1. Create project directory
mkdir monkey && cd monkey && git init

# 2. Set up OpenCode config only
/initialize-OpenCode Monkey
# → Select: NestJS + React Web + React Native
# → Creates CoFixer/monkey-OpenCode repo
# → Adds .OpenCode submodule

# 3. Add boilerplate code separately
/create-mono-repo
# → Add: nestjs-starter-kit → backend/
# → Add: react-starter-kit → frontend/
# → Add: react-native-starter-kit → mobile/
# → Generates docker-compose.yml

# 4. Create project docs manually
mkdir -p .project/{plans,memory,docs}

# 5. Initial commit
git add . && git commit -m "Initial Monkey project setup"
```
